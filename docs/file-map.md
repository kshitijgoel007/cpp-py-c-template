## C library â€” `c_lib/`

| File | Purpose |
| --- | --- |
| `c_lib/c_lib.h` | **The foreign-function contract.** Declares every exported symbol behind the `C_CORE_API` visibility macro, C status codes, and the opaque `CAccumulator` handle. C++ and Python both bind against this header. |
| `c_lib/c_lib.c` | Implementation. Only this file has access to `CAccumulator`'s internals. Compiled into a shared library with hidden visibility â€” only symbols marked `C_CORE_API` are exported. |

## Python package â€” `src/my_c_wrapper/`

| File | Purpose |
| --- | --- |
| `__init__.py` | Package entry point. Calls `require_virtualenv()` before any other import to fail fast with a readable error outside a virtualenv. Exports the public surface: `add`, `Accumulator`, `abi_version`, `version`. |
| `wrapper.py` | Hand-written idiomatic Python API. Validates inputs, calls the generated low-level ctypes functions, translates status codes to Python exceptions, and implements `Accumulator` as a context manager. |
| `_ctypes.py` | **Generated â€” do not edit.** Produced by `tools/bindings/generate_ctypes.py`. Contains `ctypes` function signatures, the `CAccumulator` opaque structure, and status code constants derived from `api.json`. |
| `_loader.py` | Native library discovery. Tries, in order: the `MY_C_WRAPPER_LIBRARY` env var, the `lib/` subdirectory of the installed package, and several build-tree paths under `build/`. Reports all attempted paths on failure. |
| `_environment.py` | Thin helper: `require_virtualenv()` raises `RuntimeError` when `sys.prefix == sys.base_prefix` (not in a virtualenv). Used by both `__init__.py` and `tools/bindings/common.py`. |
| `lib/libc_lib.dylib` _(macOS)_ | The compiled native library copied here by the cmake post-build command. Not committed to git; regenerated by `just build`. Included in wheels by `MANIFEST.in`. |
| `AGENTS.md` | Agentic instructions for the Python layer: how to add a new function, `_ctypes.py` regeneration policy, exception mapping rules. |

## Developer tools â€” `tools/`

### Binding pipeline â€” `tools/bindings/`

| File | Purpose |
| --- | --- |
| `common.py` | Shared constants and helpers for the binding pipeline: root paths, `require_virtualenv()`, `load_policy()`, `policy_headers()`. |
| `extract_api.py` | Runs Clang's JSON AST dump on the public header and emits `build/bindings/api.json`: a normalized list of every exported function's name, return type, and parameters. |
| `generate_ctypes.py` | Reads `api.json` + `bindings.toml` and generates `src/my_c_wrapper/_ctypes.py`. With `--check`, exits non-zero if the file on disk is stale. |
| `generate_abi.py` | Compiles a temporary C probe using the API definitions and runs it to record sizes, alignments, and struct field offsets of all ABI-tracked types. Emits `build/bindings/abi.json`. |
| `validate_abi.py` | Compares `abi.json` against the expected values stored in `bindings.toml`. Fails loudly if any size or offset has changed unexpectedly, preventing silent ABI breakage. |
| `package_smoke.py` | End-to-end packaging test. Creates two isolated virtualenvs and verifies that (1) an editable install loads the library and (2) a wheel install loads the library. Invoked by `just package-smoke`. |

### Other tools

| File | Purpose |
| --- | --- |
| `tools/check_environment_policy.py` | Enforces repository rules at CI and `just check` time: no `sudo` in automation files, no system Python shebangs, `require_virtualenv()` in the package, virtualenv policy mentioned in docs and instructions, every `AGENTS.md` states `sudo is prohibited`. |
| `tools/customize_template.py` | Post-copy Copier task. After `copier copy`, renames all placeholders (`MyProject`, `my_c_wrapper`, `my-c-wrapper`, `my_project`, etc.) throughout the codebase to the values the user entered. |
| `tools/template_smoke.py` | Runs `copier copy` with canned answers into a temp directory and then runs `just check` inside it to verify the template produces a working project after customization. |
| `tools/native_build_backend/backend.py` | PEP 517 build backend. On `pip wheel`, it runs cmake in `build/native-wheel/`, regenerates `_ctypes.py`, then hands off to setuptools and patches the wheel's WHEEL metadata to add a platform tag (e.g., `macosx_14_0_arm64`). |
| `tools/native_build_backend/__init__.py` | Empty. Makes `native_build_backend` a Python package importable by pip during wheel builds. |
